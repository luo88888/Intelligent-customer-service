## 三、ReAct Agent 执行流程

```mermaid
sequenceDiagram
    actor User as 👤 用户
    participant API as FastAPI
    participant ChatSvc as ChatService
    participant Memory as ConversationMemory
    participant Agent as ReactAgent
    participant Tools as Agent Tools
    participant RAG as RAG Pipeline
    participant LLM as LLM Model
    participant DB as MySQL

    User->>API: POST /api/conversations/{id}/messages
    API->>ChatSvc: execute_and_persist_stream()

    ChatSvc->>Memory: load_conversation_memory()
    Memory->>DB: 查询历史消息
    DB-->>Memory: messages[]
    Memory-->>ChatSvc: ConversationMemory

    ChatSvc->>DB: 保存用户消息

    ChatSvc->>Agent: execute_stream(query + memory.get_context())
    Agent->>Memory: add_user_message(query)
    Memory-->>Agent: 摘要 + 最近消息

    loop ReAct 循环 (Think → Act → Observe)
        Agent->>LLM: 发送消息列表 + System Prompt
        LLM-->>Agent: 思考 + 工具调用

        opt 工具调用 = rag_summarize
            Agent->>Tools: rag_summarize(query)
            Tools->>RAG: HybridRetriever → RAGSummarizeService
            RAG-->>Tools: 检索结果 + 摘要
            Tools-->>Agent: 缓存 RAG 文档 (ContextVar)
        end

        opt 工具调用 = fill_context_for_report
            Agent->>Tools: fill_context_for_report()
            Tools-->>Agent: report=True (触发 Prompt 切换)
        end

        opt 其他工具调用
            Agent->>Tools: get_weather / get_user_id / ...
            Tools-->>Agent: 工具结果
        end
    end

    Agent->>Memory: add_assistant_message(reply)
    Note over Memory: 检查是否需要增量摘要

    Agent-->>ChatSvc: yield chunks (text/rag_docs)
    ChatSvc-->>API: SSE 流式响应
    API-->>User: SSE chunks

    ChatSvc->>Memory: persist_state()
    Memory->>DB: 更新 summary_text + summarized_count

    ChatSvc->>DB: 保存助手消息
```

## 七、对话记忆增量摘要流程

```mermaid
flowchart TB
    MSG_IN["📝 新消息写入<br/>memory.add_message()"] --> CHECK{"未摘要字符数<br/>> threshold<br/>(默认 1000)?"}

    CHECK -->|否| SKIP["跳过摘要<br/>仅追加到消息列表"]
    CHECK -->|是| TRIGGER["触发增量摘要"]

    TRIGGER --> PREPARE["准备摘要输入"]
    PREPARE --> SPLIT["分割消息"]
    SPLIT --> OLD["旧消息<br/>(需要摘要的部分)"]
    SPLIT --> KEEP["最近 keep_recent_count 条<br/>(默认 6 条，保留原文)"]

    OLD --> LLM_CALL["调用摘要 LLM<br/>(DeepSeek flash, 独立配置)"]
    LLM_CALL --> PROMPT_T["使用 memory_summary_prompt.txt"]
    PROMPT_T --> NEW_SUMMARY["生成增量摘要<br/>(合并到已有摘要)"]

    NEW_SUMMARY --> STORE["更新 ConversationMemory"]
    KEEP --> STORE
    STORE --> PERSIST["持久化到 MySQL<br/>conversations.summary_text<br/>conversations.summarized_count"]

    STORE --> CTX_OUT["get_context() 输出"]
    PERSIST --> DB_STORE[("💾 MySQL")]

    CTX_OUT --> FORMAT["格式化:<br/>[摘要] 历史会话摘要: ...<br/>[最近消息]<br/>User: ...<br/>Assistant: ..."]

    style TRIGGER fill:#fff3e0,stroke:#f57c00
    style LLM_CALL fill:#e8f5e9,stroke:#388e3c
    style PERSIST fill:#eceff1,stroke:#546e7a

## 九、关键技术栈

| 层级 | 技术选型 |
|------|----------|
